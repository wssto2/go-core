I am building go-core — a shared Go library used across multiple applications, with an embedded example application (go-core-example) that demonstrates correct usage. I will attach the full packed context file containing every source file.

What go-core is
A shared Go library extracted from a production application. It contains:

apperr — typed application errors with HTTP status codes, log levels, stack location
audit — generic audit log entity, diff, and repository
auth — JWT parsing/issuing, generic User[T], Identifiable interface, middleware, RBAC
binders — HTTP request parsing, type coercion, stateless validation (JSON + multipart)
bootstrap — App (lifecycle/graceful shutdown), Container (shared infrastructure), Module interface, Config, EnvStr/Int/Bool/Duration helpers, health checks
cache — Cache interface + thread-safe InMemoryCache with ErrCacheMiss sentinel
database — GORM registry, BaseRepository, transactor, scopes, helpers, custom types (Date, DateTime, NullInt, etc.), NewRegistryFromConfigs
datatable — server-side pagination, search, filtering, ordering, views for GORM models
event — Bus interface + thread-safe InMemoryBus
go2ts — Go struct → TypeScript type generator
i18n — struct-based translator, JSON files, hot reload
logger — slog-based logger with context extraction, log rotation
middlewares — Gin middleware: CORS, security headers, request ID, logger, bind/validate (BindRequest[T], MustGetRequest[T])
resource — single-record fetcher with author resolution and sub-counts
router — pre-configured Gin engine with all core middleware
storage — FileStore interface
tenancy — tenant context, TenantAware interface, GORM scope, Gin middleware
utils — converters, helpers, pipeline
validation — struct-tag-based validator, per-instance rule registry
web — Gin response helpers (JSON, Created, NoContent, Paginated), file upload, GetParamInt
